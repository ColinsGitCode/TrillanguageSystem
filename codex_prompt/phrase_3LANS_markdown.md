description: 输入短语（中文/英文/日语）并返回另外两种语言的翻译。
argument-hint: 'phrase'
---

$ARGUMENTS
- phrase: 待识别的短语，可包含空格或特殊字符。
  - 如果短语存在拼写和错误，请先分析并修正为正确的短语，然后按照正确的短语内容，进行之后的任务

**范围**
- 仅在中文、英文、日语三种语言中判断输入所属语言。
- 对剩余两种语言分别提供简洁的翻译与一句话解释。
- 如无法确定语言，需明确说明并给出可能性判断。

**步骤**
1. 读取 `phrase` 并判断其主要语言（中文/英文/日语），不用说明判断依据。
2. 将原短语的意思翻译为其余两种语言，每种语言输出：`翻译` + `一句话解释`。
  - 其中，英文和日语的解释之后，增加两个例句
    - 例句的风格一定要是日常的口语风格，这非常重要
3. 若检测结果不确定，输出“无法判定”并给出最可能的语言排名。
4. 将最终输出保存为 Markdown 文件，存放在 `/Users/xueguodong/Desktop/trilingual_records/<YYYYMMDD>/`（若目录不存在则创建），文件名由系统统一安全化处理。
  - YYYYMMDD 必须是当天的日期

**输出格式**
```
# <原短语>

## 1. 英文:
- **翻译**: ...
- **解释**: ...
- **例句1**: ...
  - 给出例句的简要中文翻译 (作为子列表), 例句的开头不需要有任何的说明
- **例句2**: ...
  - 给出例句的简要中文翻译 (作为子列表), 例句的开头不需要有任何的说明


## 2. 日本語:
- **翻訳**: ...
  - 日语的汉字需要进行注音，格式类似 <ruby>漢<rt>かん</rt>字<rt>じ</rt></ruby>
  - 日语中的片假名外来语需要给出对应的语言 （在括号中显示对应的英语）
  - 日语中出现的英文缩写，不要进行注音
- **解説**: ...
  - 解说的内容如果是日语，有汉字的情况下也需要进行注音
  - 日语中的片假名外来语需要给出对应的语言 （在括号中显示对应的英语)
  - 日语中出现的英文缩写，不要进行注音
- **例句1**: ...
  - 例句的内容如果是日语，有汉字的情况下也需要进行注音
  - 日语中的片假名外来语需要给出对应的语言 （在括号中显示对应的英语)
  - 日语中出现的英文缩写，不要进行注音
    - 给出例句的简要中文翻译 (作为子列表) 例句的开头不需要有任何的说明

- **例句2**: ...
  - 例句的内容如果是日语，有汉字的情况下也需要进行注音
  - 日语中的片假名外来语需要给出对应的语言 （在括号中显示对应的英语)
  - 日语中出现的英文缩写，不要进行注音
    - 给出例句的简要中文翻译 (作为子列表) 例句的开头不需要有任何的说明


## 3. 中文:
- **翻译**: ...
- **解释**: ...

## 4. 技术概念简要说明
- 如果 {{ phrase }} 是一个IT领域相关的名词，请用中文给出简要的说明
- 说明的内容控制在200字左右
```
（如原语言为其中之一，仍需在该语言栏中注明“原文”并给出必要解释。）
