# TRAIN 页选区交互 UI 验证（2026-03-09）

## 1. 验证目标

验证 `TRAIN` 页新增的三类选区动作是否可用且不破坏现有链路：

- 选区生成三语学习卡片
- 选区生成日语语法卡片
- 选区标红并持久化恢复

## 2. 验证环境

- 页面：`http://localhost:3010`
- 模型链路：`GEMINI CLI PROXY`
- 验证样本卡片：
  - 日期目录：`2026.02.26`
  - 卡片：`差不多`

## 3. 验证步骤

### 3.1 TRAIN 页选区工具条出现

步骤：

1. 打开 `差不多`
2. 切换到 `TRAIN`
3. 选中 `close enough`
4. 观察浮动工具条

结果：

- 工具条正常出现
- 按钮完整显示：
  - `✦ Generate Card`
  - `📘 语法卡`
  - `🖍 标红`

### 3.2 TRAIN -> 选区生成三语卡

步骤：

1. 在 `TRAIN` 页选中 `close enough`
2. 点击 `✦ Generate Card`
3. 观察顶部队列状态条

结果：

- 任务成功入队
- 队列状态显示：
  - `#1 [三语] close enough`
  - `运行中 1`
- 任务最终成功完成

### 3.3 TRAIN -> 选区生成语法卡

步骤：

1. 在 `TRAIN` 页选中 `もう少しで終わる`
2. 点击 `📘 语法卡`
3. 在第一条任务运行期间观察排队情况

结果：

- 队列显示串行执行：
  - 第一条运行中

### 3.4 TRAIN -> 标红持久化

步骤：

1. 在 `TRAIN` 页选中 `almost there`

3. 刷新页面

结果：

- 刷新后仍能恢复
- 当前已确认恢复的高亮：
  - `almost there`
  - `もう少しで終わる`

### 3.5 持久化后控件可用性

在恢复高亮后的 `TRAIN` 页继续检查：

- `重新生成训练包` 按钮存在
- `显示答案` 按钮存在
- 训练题答案仍可正常切换

结果：

- 说明高亮恢复没有破坏 `TRAIN` 页交互控件

## 5. 验证结论

本轮结论：

1. `TRAIN` 页已具备与 `CONTENT` 页一致的选区动作能力
2. 生成动作已正确接入现有后台队列，且不打断当前浏览上下文

4. 高亮恢复后不会破坏 `TRAIN` 页已有按钮行为
