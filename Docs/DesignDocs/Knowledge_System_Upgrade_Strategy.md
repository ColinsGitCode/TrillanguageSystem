# Three LANS // CodeX 知识系统深度升级方案（v2）

**更新时间**：2026-03-03  
**适用范围**：三语学习卡 + 日语语法卡 + 可观测与评审链路

---

## 0. 升级目标

将系统从“按日期存储的卡片库”，升级为“可检索、可比较、可回溯、可进化”的跨语言知识系统，服务日语/英语双语学习者的长期积累与精准复习。

---

## 1. 当前基线（基于运行库扫描）

数据源：容器内 `/data/trilingual_records`（非仓库本地副本）

- 卡片总量：`265`
- 类型分布：`trilingual=260`，`grammar_ja=5`
- 模型分布：`gemini=261`，`local=4`
- 日期覆盖：`34` 天
- 可观测覆盖：`observability_metrics=265`（全覆盖）
- 质量分（系统评分）均值：`94.08`（中位 `96`）
- 重复短语组：`4` 组
- 音频缺失：`1` 张卡存在缺失（ruby 污染文件名导致）
- 历史字段缺口：`source_mode=null` 仍有 `231` 条

### 当前主要瓶颈

1. **语法卡占比低**：语法-例句闭环尚未形成规模。  
2. **语义边界缺失**：近义词/易混词未形成结构化对照。  
3. **知识组织方式偏时间线**：缺少主题与认知维度入口。  
4. **历史元数据不完整**：影响行为分析、实验对比与自动推荐。

---

## 2. 系统进化主线（按优先级）

## 主线 A：数据治理与索引底座（P0）

### A1. 结构化扩展
- 新增（或完善）结构：
  - `synonym_groups` / `synonym_items`
  - `grammar_patterns` / `grammar_refs`
  - `knowledge_clusters` / `card_tags`
- 目标：为语义边界、语法闭环、主题聚类提供统一数据底座。

### A2. 历史数据修复
- 回填 `source_mode`（input/selection/ocr 等）。
- 清理重复项与脏样本（同义重复、标题重复、异常文件名）。
- 修复音频任务中 `ruby` 污染文件名的问题。

### A3. 基线指标固化
- 固化每日日志：
  - 新增卡片数、重复率、音频可用率、字段完整率。
- 指标写入 DB，支持 Dashboard 趋势图。

---

## 主线 B：词义边界系统（Semantic Mapping，P1）

### B1. 反向索引
- 以中文释义为锚点，聚合同译词/近义词（英/日）。

### B2. 差异矩阵
- 维度：语气（Tone）/ 场景（Register）/ 搭配（Collocation）。
- 结果：给每个候选词生成“可用边界说明”。

### B3. 前端呈现
- 卡片详情新增“易混淆词雷达”。
- 每个词附“边界标签”（Formal / Casual / Technical / Critical）。

---

## 主线 C：语法-例句双向闭环（P1）

### C1. 语法模式抽取
- 从三语卡日语例句抽取语法模式（pattern）。

### C2. 双向关联
- 语法卡 -> 全库真实例句池。  
- 三语卡句子 -> 对应语法卡详情。

### C3. 学习可视化
- 语法卡新增覆盖度与掌握进度（按日期、按使用次数）。

---

## 主线 D：知识集群与复习编排（P2）

### D1. 自动主题聚类
- 每卡打 1~3 个主题标签（例如：AI/Tech、沟通表达、抽象逻辑）。

### D2. 领域视图（Bento）
- 提供“按主题学习/复习”入口，而非仅按日期浏览。

### D3. 个性化复习
- 支持按“质量分 + 遗忘风险 + 使用频率”自动生成复习队列。

---

## 3. 面向学习者可直接产出的能力

系统可自动产出以下学习资产：

1. **总览总结**：主题覆盖、近期重点、薄弱区域。  
2. **对照索引**：英-日-中双向检索表。  
3. **语法索引**：语法点 -> 真实例句 -> 最近出现时间。  
4. **混淆词清单**：近义词边界差异说明。  
5. **复习计划**：7/14 天按优先级自动编排。

---

## 4. 执行节奏（建议）

### Phase 1（1~2 周）
- 完成主线 A（数据治理+索引底座）
- 交付：修复脚本 + 指标面板 + 数据完整率报告

### Phase 2（2~3 周）
- 完成主线 B + C（语义边界 + 语法闭环）
- 交付：易混淆词雷达 + 语法双向跳转

### Phase 3（1~2 周）
- 完成主线 D（主题聚类 + 复习编排）
- 交付：Bento 领域视图 + 个性化复习队列

---

## 5. 验收指标（最小可行）

- 字段完整率（source_mode 等）`>= 98%`
- 音频可用率 `>= 99.5%`
- 重复短语率下降 `>= 50%`
- 语法卡引用到真实例句的覆盖率 `>= 80%`
- 易混淆词解释可用率（人工可读）`>= 90%`

---

## 6. 当前结论

系统已经具备“可观测 + 可评审 + 可扩展”的基础。  
下一阶段不应只继续堆卡片数量，而应优先建设**语义边界、语法闭环、主题聚类**三大知识能力，使学习收益从“记录型”转向“认知型”。
