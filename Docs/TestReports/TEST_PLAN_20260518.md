## 2. 五层测试金字塔

```
        L4 真实链路  (manual / opt-in CI, 真 Gemini + TTS + OCR)
            ↑
        L3 浏览器 E2E (Playwright, isolated server :3310)
            ↑
        L2 路由集成  (supertest, 内存 SQLite, E2E_TEST_MODE=1)
            ↑
        L1 单元      (node:test, pure / 内存 SQLite, ~1s)
            ↑
        L5 启动冒烟  (bash + curl, server boot + health probe)  ← 横切
```

每层职责：

| 层 | 工具 | 范围 | 何时跑 |
|---|---|---|---|
| **L1 单元** | node:test + better-sqlite3 `:memory:` | 服务/域模块/纯函数；**禁止网络**；测试用 `process.env.LOG_SILENT=1` | 每次 commit / PR |
| **L2 路由集成** | supertest + 内存 SQLite + E2E_TEST_MODE=1 + `dbService.truncateAllForTests()` per test | 单条 HTTP 请求穿过 middleware → route → service → DB → response；mock 掉出站 Gemini/TTS/OCR | 每次 commit / PR |
| **L3 浏览器 E2E** | Playwright（chromium）+ isolated server :3310 | 真实浏览器交互；server 是 E2E 模式，`/api/generate` 走 fixture 不调 Gemini | 每次 PR（CI），夜间全量 |
| **L4 真实链路** | Playwright + `RUN_REAL_GEMINI_E2E=1` | 真 Gemini + 真 TTS（如 endpoint 可用）；只覆盖 1-2 个 golden flow | 周度 / 发布前 |
| **L5 启动冒烟** | bash + curl | 拉起 server → `/api/health` → 一组 GET 端点 → exit 0 | post-deploy / 本地手动验证 |

## 4. L2 路由集成测试（最大缺口，先做）

### 4.1 基础设施

新建 `tests/integration/`：

```
tests/integration/
├── _harness.js        # 创建测试 app（不监听端口），暴露 supertest agent
├── _fixtures.js       # 共享数据 fixture（card / job / experiment）
├── generate.test.js
├── ocr.test.js
├── generationJobs.test.js
├── health.test.js
├── history.test.js
├── dashboard.test.js
├── review.test.js
├── knowledge.test.js
├── training.test.js
├── files.test.js
├── misc.test.js
└── testReset.test.js
```

`_harness.js` 要点：
- `process.env.DB_PATH = ':memory:'`、`E2E_TEST_MODE = '1'`、`LOG_SILENT = '1'`
- 用 `app` export（不调 `listen()`），supertest 接住
- 每个 test 开始 `dbService.truncateAllForTests()`

### 4.2 每个路由的覆盖矩阵

| 路由文件 | Happy path | 失败分支 | 边界 |
|---|---|---|---|
| `generate.js` | 200 + 完整 response shape（E2E fixture） | 400（缺 phrase）/ 422（validation）/ 429（throttle） | `X-Generation-Job-Worker:1` 跳过 throttle |
| `ocr.js` | tesseract（mock） / local（mock）/ auto fallback | 400（无 image / 未知 provider） | E2E fixture 返回固定文本 |
| `generationJobs.js` | create → list → retry → cancel → event log | 重复 enqueue / 不存在的 jobId | stale recovery（startup hook） |
| `health.js` | overview 整合健康 | gemini auth subroutes | E2E mode 标志 |
| `history.js` | 分页 / 搜索 / provider+cardType+date 筛选 / recent | 空集 / FTS 语法 | 大分页边界 |
| `dashboard.js` | 各聚合端点 shape | 空 DB | — |
| `review.js`（11 routes） | campaign 创建 + 5 维评分 + 列表 | 不存在的 campaign | eligibility 流转 |
| `knowledge.js`（17 routes） | 6 个 task 类型 enqueue → status → result；overview / index / synonym / grammar / cluster / issues 各 GET | 不存在的 job / 越界 limit | E2E fixture job 路径 |
| `training.js`（5 routes） | backfill summary + candidate list + run | 不存在的 generation | E2E 跳过真 Gemini |
| `files.js` | folders 列表 / highlights upsert + delete / records/by-file | 路径穿越保护 / 不存在的文件 | source_hash 版本 |
| `misc.js` | experiments export / DELETE /api/records/:id | 不存在的 id | — |
| `testReset.js` | E2E 模式下 truncate + records 清空 | **生产模式 404**（在 _harness 关 E2E_TEST_MODE 单独测一次） | — |

### 4.3 工时估算

每路由 2-4 个测试，平均 30 分钟一组 → 约 1-1.5 天补齐全部 13 个路由（~50 测试新增）。

## 7. L5 启动冒烟脚本

新建 `scripts/smoke.sh`：

```bash
#!/usr/bin/env bash
set -euo pipefail

PORT="${PORT:-3987}"
node server.js &
SRV=$!
trap "kill $SRV 2>/dev/null" EXIT
sleep 1.5

curl -fsS "http://127.0.0.1:$PORT/api/health" >/dev/null
curl -fsS "http://127.0.0.1:$PORT/api/history?page=1&limit=1" >/dev/null
curl -fsS "http://127.0.0.1:$PORT/api/dashboard/summary" >/dev/null
echo "smoke OK"
```

预期：< 3 秒。post-deploy / 本地变更后手动跑。

## 8. CI 集成（推荐落地）

新建 `.github/workflows/ci.yml`：

```yaml
name: CI
on: [push, pull_request]
jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with: { node-version: '20' }
      - run: npm ci
      - run: npm run lint   # 0/0 基线
      - run: npm test       # 238/238

  e2e:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with: { node-version: '20' }
      - run: npm ci
      - run: npx playwright install --with-deps chromium
      - run: npm run test:e2e:smoke
      - run: npm run test:e2e:pages
      - run: npm run test:e2e:gemini-sanitize
```

真实 Gemini E2E 用 workflow_dispatch 手动触发。

## 12. 风险与注意

- **测试环境隔离**：所有非 L4 层必须 `E2E_TEST_MODE=1` + `:memory:` DB + 临时 RECORDS_PATH，**任何测试不得写真实 `./data/trilingual_records.db`**。
- **flaky 来源**：CURRENT_TIMESTAMP 秒级精度（同秒插入 ID 排序不稳）、tts/ocr 网络抖动、gemini quota 限速。集成层用 mock，e2e 用 fixture，可控。
- **数据夹带**：`scripts/startE2EServer.sh` 每次清空 `.tmp/e2e`；新 spec 必须 beforeAll resetServerState。

## 13. 后续维护
